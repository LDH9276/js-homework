# 16기 이동헌 JavaScript 2주차 과제

## 과제 수행 과정
- 해당 과제를 최대한 길게 풀어 쓰려 했으나, if / switch 문으로 작성했습니다.
- if문을 없이 연산자로만 하려니 시간이 없어 완료하지 못했습니다. 연산자를 좀 더 잘 써야 할 것 같습니다.
- 결국 if문으로 3, 4, 5번을 해결했습니다.
- 또한, 삼항 연산자를 자주 써왔다 보니 익숙해져서 무의식적으로 자주 사용하게 되었고, 오히려 더 편했습니다.
- 다만 이것도 줄여야 할지 말아야 할지 고민이네요.
- 아예 배운 과목대로 하라니 정말 막히더라고요. 설계하는 방식에 대한 고차원적인 고민이 필요할 것 같습니다.

## 테스트 코드 작성
``` javascript
console.log(greetingMessage('이동헌', true))
// 결과값 : '🌟 VIP 이동헌님, 특별한 혜택이 준비되어 있습니다!'
console.log(greetingMessage('이동헌', false))
// 결과값 : '안녕하세요, 이동헌님! 즐거운 쇼핑 되세요.'

console.log(deliverCalc('0', '제주'))
// 결과값 : 0
console.log(deliverCalc('50000', '서울'))
// 결과값 : 0
console.log(deliverCalc('3200', '서울'))
// 결과값 : 3000
console.log(deliverCalc('3200', '도서'))
// 결과값 : 6000

console.log(checkPassWordLength('1234567'))
// 결과값 : false
console.log(checkPassWordLength('1234567890'))
// 결과값 : true
console.log(checkPassWordLength('123456789012345789012'))
// 결과값 : false

console.log(calcPoint('vip', '100050원'))
// 결과값 : 5002

console.log(movieCharge('IMAX', true, '1명'))
// 결과값 : 16000
console.log(movieCharge('Imax', true, NaN))
// 결과값 : 0
```

## 과제 코드 리뷰
- 보통 상태를 볼때 is(변수명)을 자주 썼습니다. 이경우 true, false 값을 정하기 편하더라고요
- 그 외에는 변수짤 때 신경을 써야할 것 같습니다. 알기 힘든 변수도 많더라고요.

### greetingMessage() 리뷰
- 두가지 버전으로 만들었습니다. 하나는 연산식, 하나는 조건문
- 두 버전의 차이는 return에서 다른데, 저는 오히려 삼항연산자로 쓴 것이 더 깔끔하지 않나 싶습니다.
- 다만 이후에 배울거지면 nullish와 잘못된 값이 올 때 쓰는 `??=` 는 PHP와 동일해서 놀랐습니다.

### deliverFee() 리뷰
- 도저히 연산자로 짜기에는 애매해서 삼항 연산자로 짰습니다. 아래 if문 보다는 깔끔하더라고요.
- if문으로 작성한 `deliverCalc()`도 있습니다만, 다시 검색해서 막히는 부분만 수정한 `deliverFee()`이 더 간결했습니다.

### checkPassWordLength() 리뷰
- 자리값 단위다보니 length 메서드를 썼습니다. 이게 더 낫더라고요.
- 이 문제는 연산자만 이용하면 되는 쉬운 문제였습니다.

### calcPoint() 리뷰
- switch 짰습니다. if문으로 하면 if(조건) / else if(조건)이라 가독성이 나쁘므로요.
- case 안에 return을 써도 된다지만, break를 걸어서 안전하게 했습니다.
- 다만 소숫점으로 끝나는 경우도 있어서 뭐지? 했는데 소숫점을 올렸습니다.

### movieCharge() 리뷰
- 배열로 접근한다는 것을 알고나서 바꾸니 훨씬 낫더라고요.
- 다만 조조할인 여부일때는 true, false로 접근하게 했고 삼항 연산자를 통해 해결했습니다. 연산자로도 푸는 방법을 연습해봐야 될 것 같습니다.
- 그리고 마지막에 parseInt(person)에서 NaN이 올수도 있어서 검색해보니 || 0 으로 처리가 가능한 것을 알았습니다.
