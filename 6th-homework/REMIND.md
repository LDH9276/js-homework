# 16기 이동헌 JavaScript 4주차 과제

## 과제 수행 과정
- 기존의 클릭 이벤트를 복습한다 생각하고 작성했습니다.
- 가장 먼저 할 것은 시맨틱 코드로 짠 후에 지난 과제 처럼 DOM 랜더링을 시키도록 했습니다.
- 각 안의 내용 클래스와, 접근성을 위해 `arid-pressed`를 사용했습니다.
- `div`가 아닌 `label`로 접근성 측면을 명시했습니다.

## 테스트 코드 작성
``` bash
bun dev
```

## 과제 코드 리뷰
- 마찬가지로 컴포넌트로 속성을 넣어 랜더링하도록 했습니다.
- 덤으로 추가적으로 배열은 `map()`함수로 나오는 것을 배웠습니다.
- 이번에는 상당히 간단하게 제작했지만, 가격 같은 부분은 if문으로 하는게 간단하긴 한데 삼항 연산자를 써보았습니다.
- 더불어 기존 코드를 복습하며 아래 플랜 출력 부분도 만족스럽게 수정했습니다.
- `div`가 아닌 `button`으로 하다 버튼 속성에는 `ol, ul, dl`을 넣으면 안되는 것을 알았습니다. 그래서 `label`과 `input`요소로 바꾸었습니다.
- `label`에서 탭 컨트롤이 아니라 키보드 컨트롤이 가능해 예제에 맞게 `button`요소로 다시 수정했습니다.

## 회고
- 접근성 있게 생각하는 시간이 많아져서 만족스럽습니다.
- 기본적인 컴포넌트에 좀 더 복잡한 기능을 넣기 시작해서 만족스럽습니다.
- 앞으로도 이런 방향으로 제이쿼리가 아닌 순수 ES 자바스크립트로 하는 것이 더 편해졌습니다.
- 다만 설계를 여러번 뜯어 고쳐야 해서 난감한 과제였던 것 같습니다. 앞으로 시맨틱 구조부터 다시 생각해 봐야겠습니다.
